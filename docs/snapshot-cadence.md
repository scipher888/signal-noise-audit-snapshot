# Snapshot Cadence and Versioning

**Status:** public-snapshot candidate  
**Snapshot date:** 2026-05-13

This repository uses milestone snapshots, not a raw live mirror of internal working files.

## Why milestone snapshots

A raw repository would expose too much: private working notes, unpublished drafts, internal strategy, personal context, and operational details that are not part of the public standard.

Milestone snapshots give readers a stable public artifact without pretending that every internal draft belongs in public.

## Versioning

- **v1.0** — first public snapshot of the editorial constitution and corrections / changes surface.
- **v1.x** — material clarification, added section, or meaningful wording change that preserves the same overall constitution.
- **v2.0** — structural change to the trust stack, editorial authority model, claim-gating system, or AI process framing.

## Snapshot triggers

A new public snapshot should be created when any of the following occurs:

1. a substantial section-level change to the constitution;
2. a new constitutional section is added;
3. a locked editorial decision changes scope or public accountability;
4. a tracked claim is corrected, retired, or materially narrowed;
5. public methodology changes in a way a reader would reasonably expect to know;
6. quarterly review finds meaningful drift between the public snapshot and current practice.

## Changelog convention

Each new snapshot should include:

- version number;
- date;
- one-paragraph summary;
- bullet list of changes;
- links to affected public artifacts;
- whether the change is a correction, clarification, scope narrowing, or new commitment.

## Archive rule

Old snapshots should remain accessible. Do not silently overwrite public standards.

## Raw repository upgrade path

A fully public raw repository is not currently planned.

It may become appropriate if:

- readers or auditors repeatedly ask for finer-grained history;
- a specific issue depends on history that milestone snapshots cannot support;
- an external auditor or curator asks for underlying files;
- the redaction burden becomes manageable.

Until then, the public standard is milestone snapshots plus corrections / changes.
